Prerequisites

Kiwi is a bridge system to map data from Emu into Solr. It is therefore required that you have a functional Emu server and Apache Solr server up and running first.

You must have a server running PHP 5.2 or later on which to install Kiwi. That does not have to be the same server that Emu or Solr are on. The following PHP extensions are expected:

  • ext/pcntl - PHP Process Control
  • ext/tidy - HTML/XML formatter. This extension is optional but recommended for more useful debug output.

Download

Download the latest stable release of Kiwi from the links on the front page. uncompress the download and place the code somewhere on your server.

Configure Solr

  1. Set up a new, empty Solr core. Consult the Solr documentation for how to do so.
  2. Copy the contents of the solr-conf directory to the conf directory of your Solr core.
  3. Restart your Solr server so that your changes will take effect.

Prepare your configuration file

All control of Kiwi is managed via an XML configuration file. Each configuration file defines one mapping from Emu to a Solr core.

A Kiwi configuration file must conform to the RelaxNG schema defined in the config-schema.rng file

See the included config-example.xml file for an example of the configuration format. It is heavily documented.

Depending on your use case, you may write a single configuration file or put multiple in a directory together. Kiwi is able to batch-process an entire directory's worth of configuration files at once.

If necessary, configuration files support the XInclude standard. That is, portions of a config file that you want to write in an separate, shared file may be referenced by any number of configuration files. That can be especially useful for field type configuration.

Profit!

You are now ready to Run Kiwi.
